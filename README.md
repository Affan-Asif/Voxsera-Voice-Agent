# AI Voice Agent Platform
LIVE DEMO: https://voxsera-voice-agent-production.up.railway.app/

An AI-powered voice receptionist platform for SMBs. Businesses can configure a custom AI persona and voice, connect Google Calendar and Google Sheets, and receive a Twilio phone number answered by a low-latency, full-duplex AI voice agent with barge-in support. Powered by Gemini for reasoning and function calling, Deepgram and Sarvam AI for speech processing, the agent schedules appointments, checks availability, updates Google Sheets, makes automated reminder calls 30 minutes before appointments, and stores transcripts, sentiment, and call history in Supabase, all through a React dashboard.

Tech Stack: FastAPI, React (Vite), Twilio Media Streams, Gemini, Deepgram, Sarvam AI, Google Calendar & Sheets, Supabase.

Assuming you've already cloned/downloaded the project, these are the basic steps to run it.
1. Start the backend

```bash
cd backend

# Create a virtual environment
python -m venv venv

# Activate it
# Windows
venv\Scripts\activate

# macOS/Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Copy environment file
copy .env.example .env    # Windows
# or
cp .env.example .env      # macOS/Linux
```

Fill in all the required API keys in `backend/.env`, then run:

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

---

2. Expose your backend (for Twilio & Google OAuth)

Install and run ngrok:

```bash
ngrok http 8000
```

Copy the HTTPS URL, for example:

```
https://abcd1234.ngrok-free.app
```

Set it as:

```
BASE_URL=https://abcd1234.ngrok-free.app
```

in your `.env`.

---

3. Configure Twilio

In your Twilio phone number settings:

**Voice Webhook**

```
https://<your-ngrok-url>/twilio/voice
```

Method:

```
POST
```

---

4. Configure Google OAuth

In Google Cloud Console:

* Enable **Google Calendar API**
* Enable **Google Sheets API**
* Create an OAuth Client ID
* Add the redirect URI:

```
https://<your-ngrok-url>/api/google/callback
```

Copy the Client ID and Secret into `.env`.

---

5. Set up Supabase

* Create a Supabase project.
* Open the SQL Editor.
* Run `backend/schema.sql`.
* Copy the Project URL and Service Role Key into `.env`.

---

6. Start the frontend

```bash
cd frontend

npm install

npm run dev
```

Open:

```
http://localhost:5173
```

---

7. Test everything

* Open the dashboard.
* Connect your Google account.
* Configure your AI agent.
* Call your Twilio number.
* The AI should answer, speak, book appointments, and log calls.

---

Full-Duplex & Low-Latency Voice Pipeline

The platform uses Twilio Media Streams to establish a bidirectional audio connection, allowing the caller and AI to speak simultaneously. Caller audio is streamed in real time to Deepgram for speech recognition, while AI responses are generated by Gemini and converted to speech using Deepgram (or Sarvam AI) with minimal delay. If the caller interrupts, voice activity detection immediately stops the AI's speech (barge-in) and gives control back to the caller, creating a natural conversation. Streaming TTS and optimized endpointing ensure fast responses with near real-time turn-taking.
